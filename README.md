# Seizure Seeker

Seizure Seizure is a browser-based EEG seizure-like event detector that runs entirely client-side. It processes `.edf` (European Data Format) and `.csv` EEG files, computes signal energy using overlapping RMS windows, and identifies candidate seizure segments via robust statistics.

## Overview

Seizure Seeker provides an instant, privacy-first tool to detect seizure-like segments in EEG recordings. All processing is done in-browser‚Äîno data leaves your machine.

## Features

- **Input**: EDF/EDF+ & CSV  
- **On-the-fly parsing**: Papa Parse for CSV, jsEDF for EDF  
- **Robust baseline**: median + MAD of the quietest 50% of frames  
- **Parameters**:
  - Z‚Äëscore threshold (`z`)
  - Minimum duration (`minDur`)
  - Merge gap (fixed at 5s)  
- **Instant results**: renders a dynamic table in seconds  
- **Privacy**: EEG never leaves browser

## Algorithm Details

**1. Windowing**  
- 1‚ÄØs RMS frames with 50% overlap

**2. RMS Energy**  
\`\`\`  
rms[i] = sqrt((1/W) * Œ£_{j=0..W-1} x[i*hop + j]^2)  
\`\`\`

**3. Baseline**  
- Sort all `rms`, take lowest 50% ‚Üí array `quiet`.  
- \`median = quiet[mid]\`  
- \`MAD = mean(|quiet - median|)\`

**4. Threshold**  
- \`thr = median + z * MAD\`  
- Frames above \`thr\` are flagged.

**5. Segmentation**  
- Group consecutive hot frames ‚Üí initial segments.  
- Discard segments \< \`minDur\`.

**6. Merge Gaps**  
- Merge segments separated by ‚â§ 5‚ÄØs.  
- Final filter: discard \< \`minDur\`.

## Parameter Tuning

| Parameter        | Default | Description                                              |
|------------------|---------|----------------------------------------------------------|
| **z**            | 5       | Multiples of MAD above median to flag a frame.          |
| **minDur**       | 15‚ÄØs    | Minimum segment length to accept.                       |
| **mergeGap**     | 5‚ÄØs     | Maximum silences to bridge adjacent detections.         |

Adjust to trade off sensitivity vs. specificity.

## Performance

- **Complexity**: O(N) for RMS + segmentation  
- **Memory**: O(N/2) for RMS array  
- **Benchmark**: ~0.1‚ÄØs for a 1‚ÄØhr, 256‚ÄØHz recording on a modern laptop
